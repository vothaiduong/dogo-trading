# Dogo Corporation — WordPress site

Custom WordPress theme + local Docker stack + one-command deploy to a Vultr VPS.

- **Theme**: `wp-content/themes/dogo-corporation/` — classic PHP theme, multilingual (EN / VI / JA), UI inspired by [cursor.com](https://cursor.com) (warm cream surface, oklab borders, three typographic voices).
- **Content**: based on [dogo-trading.com](https://dogo-trading.com) — cross-border eCommerce for authentic Japanese products (Collectibles, Beauty, Kitchenware, Proxy procurement).

```
.
├── DESIGN.md                       # Cursor-derived design tokens (reference)
├── docker-compose.yml              # local WP + MariaDB + phpMyAdmin
├── Makefile                        # `make help` for everything
├── .env.example                    # copy to `.env` for local + prod creds
├── deploy/
│   ├── provision-vultr.sh          # one-shot Ubuntu 22/24 server bootstrap
│   ├── deploy.sh                   # rsync theme to PROD_HOST

    ├── template-parts/             # hero · stats · services · about · work · cta · contact
    ├── inc/                        # theme-setup · enqueue · template-tags · i18n
    ├── assets/css/main.css         # full design system
    ├── assets/js/main.js           # mobile menu · sticky header · reveal-on-scroll · parallax
    ├── assets/images/              # hero, services, about, logo (sourced from dogo-trading.com)
```

## Local development

**Requirements**: Docker Desktop (or Colima / OrbStack) + GNU Make.

```bash
cp .env.example .env       # optional — defaults work
make up                    # start WP on http://localhost:8080
make install               # one-time: install WP core, activate theme, install Polylang
```

Open:

- Site: <http://localhost:8080>
- Admin: <http://localhost:8080/wp-admin>  (admin / admin)
- DB GUI: <http://localhost:8081>

The theme directory is bind-mounted, so file changes show up on refresh — no rebuild needed.

### Common make targets

```
make help            # list everything
make logs            # tail WP + DB logs
make restart         # restart Apache (after PHP edits that need a reload)
make wp ARGS="..."   # run any wp-cli command, e.g. make wp ARGS="plugin list"
make shell           # bash inside the WP container
make mysql           # MySQL prompt
make reset           # wipe DB and start fresh (asks for confirmation)
make package         # build dist/dogo-corporation.zip for manual upload
```

### Switching language locally

The theme reads `?lang=vi` / `?lang=ja` and loads the matching `.mo` file out of the box (good for previewing translations before installing Polylang).
Once Polylang is installed (`make install` does this), use its admin UI to assign per-page translations and the language switcher in the header will use Polylang's URLs automatically.

## Deploying to Vultr

Full walkthrough in [`deploy/DEPLOY.md`](deploy/DEPLOY.md). Short version:

1. Spin up an Ubuntu 22.04 / 24.04 droplet on Vultr, point your domain's `A` record at it.
2. Fill in `PROD_*` values in `.env`.
3. `make deploy`

That single command will:

- SSH into the server, install Nginx + PHP 8.2-FPM + MariaDB + certbot
- Create the database, install WordPress, install Polylang, install vi/ja language packs
- Configure the Nginx vhost, request a Let's Encrypt cert
- `rsync` the theme up and activate it

After the first deploy, ongoing pushes are just:

```bash
make deploy-files       # theme only
make deploy-images      # uploads only
```

## Editing the theme

- **Sections**: each homepage section is a separate file in `template-parts/`. Add a new one and reference it in `front-page.php` with `get_template_part('template-parts/yourname')`.
- **Services & stats data**: defined in `inc/template-tags.php` (`dogo_services()`, `dogo_stats()`). Filterable via `add_filter('dogo_services', ...)` if you later add a CMS panel.
- **Translations**: edit `languages/dogo-corporation-vi.po` / `-ja.po`, then `msgfmt -o NAME.mo NAME.po` (or use Poedit). The `.mo` files are what WP actually loads.
- **Images**: drop new files into `assets/images/` and reference them with `dogo_img('filename.jpg')` from PHP.
- **Design tokens**: see top of `assets/css/main.css` (`:root { --surface-200: ... }`). Mirrors `DESIGN.md`.

## Notes

- The DESIGN.md was generated by `npx getdesign@latest add cursor` and used as the source of truth for color/typography/spacing decisions. Cursor's three signature fonts are proprietary, so the theme maps them to the closest free Google equivalents: **Inter** (display/UI), **Newsreader** (editorial serif), **JetBrains Mono** (code/labels).
- The fallback language switcher in the header works with `?lang=xx` even before Polylang is installed, so you can preview localized text immediately.
- All hardcoded copy is wrapped in `__('…', 'dogo-corporation')` — `make wp ARGS="i18n make-pot wp-content/themes/dogo-corporation languages/dogo-corporation.pot"` will regenerate the POT after you add new strings.
